# Iteration 1: 

# What questions did you ask? What surprised you about their answers? If you used AI to generate the initial HTML, what did it get right? What did you have to change to match the client's needs? 

To start this project, I interviewed some students who work in the Babson First-Generation Student Success Office to understand what they actually needed. I showed them a pre-compiled list of reference sites from other colleges first-generation student offices, which helped me understand their preferences. They preferred lots of images, short sections of text, and a color palette pulled from their Instagram. One thing that surprised me was how unsure they were about the content itself. Many of their programs are new or still changing, so they couldn't give me a definitive list of what to include. That made it harder to build with real content from the start, since I wasn't sure what was allowed or accurate. The HTML structure for three pages (home/landing page, programs page, and contact page) uses semantic elements like <header>, <nav>, <main>, and <footer>, and the built in Copilot in VS Code helped with some of the initial structure. I moved forward with CSS styling (as in Iteration 2’s learning log entry) and came back to HTML later when the content I was meant to add was more clear. 

 

# Iteration 2: 

# What CSS properties did you learn? If you asked AI for help with styling, what did it suggest? Did you accept everything or modify some suggestions? 

In my external stylesheet, I applied the color scheme and font choices the client asked for: sans-serif fonts, the navy/green/mustard palette, and a layout that kept text sections short with space for images. I used Copilot to help with some of the CSS, particularly flexbox layout for the navigation and section alignment. I also tried to use Copilot to help with spacing the images, but modified that manually when I did not feel it was understanding how I wanted the images. The most difficult aspect of using my CSS stylesheet was that the longer my CSS file got, the harder it became to track down which class was affecting which element. Because of this, I often used Copilot to help find the sections I was looking for. 

 

# Iteration 3: 

# What feedback did your client give? What was the hardest change to make? What's one thing you'd do differently next time? 

The site that I showed my client for feedback was not entirely finished, but still allowed them to understand the direction I was going in. It lacked a lot of real content because I wasn’t sure until after receiving this feedback what they wanted me to include. The students liked the images and graphics, showed me the correct links to include for each program, and verified the program information. After this meeting I was also focused on cleaning up the style of the contact page, which had a lot of random italics or bold text, some inside of containers and some not. It was difficult to navigate the CSS stylesheet to make all these little changes. For the next project, I want to be sure to push for an earlier client meeting and have more organized class/ID titles for elements.
